01  项目简介与背景
项目背景
• 传统纸质考试模式存在组织成本高、阅卷效率低、成绩统计繁琐等问题
• 在线教育快速发展，对高效、公平的考试评估工具需求日益增长
• 现有在线考试系统在防作弊、题型支持、用户体验等方面仍有不足
• 本项目旨在构建一套功能完善、安全可靠的多角色在线考试平台
项目目标
• 实现管理员、教师、学生三角色分离的权限管理体系
• 支持单选、多选、判断、填空、简答、编程六种题型
• 提供题库管理、智能组卷、在线考试、自动评分全流程
• 构建防作弊机制，保障考试的公平性和严肃性
项目名称
在线考试系统
开发周期
数月（需求→设计→编码→测试→文档）
系统架构
前后端分离，多模块独立部署
用户角色
管理员、教师、学生
核心功能
题库/试卷/考试/答题/评分/统计
部署方式
支持开发环境 & 生产环境部署

流程特点
• 状态驱动：考试状态严格流转（未发布 → 已发布 → 进行中 → 已结束），防止非法操作
• 双写保障：答案同时写入 MySQL 和 Redis，确保数据不丢失，支持断网恢复
• 自动评分：单选/判断直接比对，多选排序后比对，填空/简答/编程标记为待批改
• 防作弊机制：切屏检测 + 自动交卷 + IP 记录，多层保障考试公平性

04  项目亮点与难点攻克
关键技术难点与解决方案
跨域与统一认证
问题：问题：三端前端访问不同后端端口，浏览器拦截跨域请求；三角色共用登录入口需区分路由
方案：方案：Vite proxy 开发环境代理 + Nginx 生产环境反向代理；JWT Token 含 userType 字段，前端 Pinia 存储角色状态
高并发死锁
问题：问题：200并发提交试卷时，update 顺序不一致导致数据库死锁
方案：方案：统一事务执行顺序（先 exam_record 后 answer_record）；添加数据库索引优化锁粒度
答案实时保存
问题：问题：考试过程中网络中断或浏览器崩溃导致答题数据丢失
方案：方案：MySQL + Redis 双写策略；前端 30 秒自动保存；Redis 异常时仅记录日志不影响主流程
批量导入兼容
问题：问题：Excel/Word 格式多样，特殊字符导致解析失败
方案：方案：Apache POI 解析；标准导入模板；字符过滤与转义；预览确认后正式导入
切屏检测兼容
问题：问题：不同浏览器对 visibilitychange 事件支持不一致
方案：方案：同时监听 visibilitychange 和 blur 事件；服务端接口双重校验；防抖逻辑避免误触发

2. 启动 Redis 缓存服务
